# Протокол юзабилити-теста

Согласно главе 11 §11.6.6 пояснительной записки.

## Цель

Оценить удобство использования SpectrumAI представителями целевой
аудитории (студенты химических специальностей) и собрать SUS-балл.

## Состав респондентов

3–5 человек, имеющих опыт работы с ИК-спектрами на уровне курса
органической химии. Согласие на участие — устное; идентифицирующие
данные не собираются.

## Подготовка

1. Поднять стек: `docker compose up -d`.
2. Открыть http://localhost в браузере.
3. Подготовить 3 файла-образца: 2 валидных JCAMP-DX из NIST WebBook
   (этанол, ацетон) и 1 битый файл (для проверки сообщения об ошибке).
4. Перевести интерфейс на русский язык (по умолчанию).

## Сценарии

Респондент выполняет задания самостоятельно; модератор записывает
время выполнения, количество ошибок и эмоциональную реакцию.

| № | Задание | Целевое время | Критерий успеха |
|---|---------|---------------|-----------------|
| 1 | Загрузить файл `ethanol.jdx` и получить результат идентификации | ≤ 60 с | Виден список кандидатов и групп |
| 2 | Открыть карточку соединения из топ-3 кандидатов | ≤ 30 с | Открылась страница соединения |
| 3 | Найти в базе соединение по имени | ≤ 60 с | Соединение найдено в таблице |
| 4 | Открыть историю и повторно просмотреть результат | ≤ 30 с | Кликом на запись открылась страница результатов |
| 5 | Загрузить битый файл и понять причину ошибки | ≤ 30 с | Респондент озвучивает, что файл некорректен |
| 6 | Запустить пакетную обработку 3 файлов | ≤ 90 с | Появилась сводная таблица |

## Сбор данных

После прохождения сценариев респондент заполняет
[`sus-questionnaire.md`](sus-questionnaire.md).

Модератор отмечает:

- Время выполнения каждой задачи;
- Количество отказов / необходимости подсказки;
- Свободные комментарии;
- SUS-балл (десятый шаг расчёта).

## Целевые значения

- SUS ≥ 70 — приёмочный порог (NFR-07);
- SUS ≥ 75 — целевое (NFR-07);
- 0% полных отказов (если респондент не смог выполнить задание даже с подсказкой).

## Анализ результатов

Итоговый SUS — медиана баллов всех респондентов. Все комментарии
собираются и группируются по темам (UI-проблемы, неоднозначные
формулировки, ошибки взаимодействия). Результаты заносятся в
итоговый раздел пояснительной записки (раздел 11.12).
