# Руководство пользователя SpectrumAI

Документ описывает работу с системой распознавания органических соединений по
инфракрасным спектрам. Аудитория — студенты и сотрудники химических
лабораторий, имеющие опыт работы с ИК-спектрами на базовом уровне (NFR-09,
глава 11 §11.6.6 пояснительной записки).

Перед началом работы убедитесь, что система запущена (см. `docs/INSTALL.md`).
Веб-интерфейс открывается по адресу **http://localhost** в современном
браузере (Chrome, Firefox, Edge).

---

## 1. Обзор интерфейса

При первом запуске вы попадаете на страницу «Идентификация». В левой части
размещена навигация:

- **Идентификация** — главный сценарий: загрузка спектра и получение результата;
- **Пакет** — пакетная обработка нескольких файлов;
- **База соединений** — справочник эталонных соединений с поиском и фильтрами;
- **История** — журнал ваших прошлых запросов идентификации;
- **Настройки** — параметры обработки (Top-K, Grad-CAM, нормализация и т. п.).

В верхней панели — переключатель языка (русский по умолчанию, английский
доступен в один клик).

---

## 2. Идентификация одиночного спектра (UC-01)

1. На странице «Идентификация» перетащите файл спектра в зону drag-and-drop
   **или** нажмите на неё, чтобы открыть диалог выбора. Поддерживаются:
   - **JCAMP-DX** (`.jdx`, `.dx`) — основной формат NIST, SDBS, Coblentz;
   - **CSV/TXT** (`.csv`, `.txt`) — две колонки: волновое число (см⁻¹), интенсивность.
   Максимальный размер файла — **10 МБ**.

2. Под зоной появляется превью загруженного файла. Если вы выбрали не тот
   файл, нажмите крестик «×» и загрузите другой.

3. В блоке «Параметры обработки» можно установить:
   - **Top-K** — сколько кандидатов вернуть в списке (1–50; по умолчанию 10);
   - **Grad-CAM** — включить тепловую карту активаций для top-1 группы.

4. Нажмите **«Идентифицировать»**. На странице покажется индикатор прогресса
   с шагами «Парсинг → Предобработка → ML-инференс → Готово».

5. По окончании вы будете перенаправлены на страницу «Результаты».

> Если файл не подходит под требования (битый JCAMP, неверный диапазон, NaN
> в данных), система покажет красное сообщение с указанием места ошибки —
> например, «Отсутствует ##XUNITS» или «Нечисловое значение в строке 7».

### Кнопка «Использовать образец»

Если у вас нет под рукой реального спектра, нажмите **«Использовать образец»**.
Система загрузит тестовый этанол-подобный спектр и сразу начнёт идентификацию.

## 3. Чтение результатов

Страница «Результаты идентификации» содержит четыре блока:

### 3.1. График спектра

Интерактивный график 400–4000 см⁻¹ (Plotly):
- наведение курсора показывает волновое число и интенсивность;
- зум — выделение прямоугольника мышью; двойной клик — сброс.
- кнопка «Поглощение / Пропускание» переключает представление оси Y.

Если вы выбрали функциональную группу (см. §3.2), поверх спектра рисуется
**тепловая карта Grad-CAM** — красные области показывают, какие участки
спектра модель использовала для предсказания этой группы.

### 3.2. Функциональные группы

Цветные бейджи: каждое предсказание из 25 групп с вероятностью.
- **Зелёный** (≥ 0.80) — высокая уверенность;
- **Жёлтый** (0.50–0.80) — умеренная уверенность;
- **Серый** (< 0.50) — группа **не** прошла порог (модель не считает её
  присутствующей).

По клику на бейдж активируется Grad-CAM для этой группы. Под списком —
переключатель «Показать все группы / Только предсказанные».

### 3.3. Кандидаты-соединения

Карточки в порядке убывания similarity score:
- **Структурная формула** (RDKit-рендеринг SVG);
- Название, формула, CAS-номер;
- **Score** — косинусное сходство в эмбеддинг-пространстве (1.0 — полный
  совпадение, 0.0 — полное несоответствие);
- **Согласован с предсказанием (Jaccard X)** — зелёный значок, если набор
  функциональных групп кандидата совпадает с предсказанными группами.

Кнопка **«Подробнее»** на карточке открывает полную страницу соединения
(IUPAC-имя, InChI, молекулярный вес, ассоциированные спектры).

### 3.4. Действия

- **«Сохранить отчёт»** — генерирует PDF с шапкой, графиком спектра,
  таблицами групп и кандидатов, опциональной Grad-CAM. Файл скачивается
  под именем `identification-<request_id>.pdf` (UC-05).
- **«Новый поиск»** — возвращает на страницу идентификации.

Кнопки «Поделиться» и «Добавить в базу» зарезервированы для следующих фаз
проекта (фаза 2).

## 4. Поиск по базе соединений (UC-04)

Страница «База соединений»:
- В поле **Поиск** введите название, формулу или CAS-номер. Поиск
  выполняется по подстроке (case-insensitive), с задержкой 300 мс после
  последнего ввода.
- Список **«Функциональные группы»** — щёлкая бейджи, вы добавляете фильтр
  (показывать только соединения, содержащие выбранные группы). Кнопка
  **«Сбросить»** убирает все фильтры.
- Таблица с результатами: имя, формула, CAS, SMILES. Кнопка **«Подробнее»**
  открывает полную карточку соединения.
- Пагинация внизу: **Предыдущая / Следующая**, по 20 записей на странице.

---

## 5. История запросов (UC-08)

Страница «История» содержит все ваши прошлые запросы:
- Колонки: дата/время, имя файла, статус (успех / ошибка), время
  обработки, top-предсказанные группы.
- Клик по строке открывает страницу результатов того запроса
  (`/identify/results/<id>`).

> На фазе 1 история хранит только метаданные запроса и предсказанные
> группы. Сам спектр, кандидаты и Grad-CAM **не сохраняются** — они
> доступны только в текущем сеансе. Перейдя по ссылке из истории, вы
> увидите соответствующее сообщение.

---

## 6. Пакетная обработка (UC-06)

Страница «Пакет»:
1. Перетащите несколько файлов в зону drag-and-drop (или используйте кнопку
   «Выбрать файлы»). Поддерживается **до 20 файлов**, общий размер ≤ 500 МБ.
2. Файлы появятся в очереди со статусом «В очереди».
3. Нажмите **«Запустить»**. Каждый файл обрабатывается последовательно;
   при ошибке показывается её причина, остальные файлы продолжают обработку.
4. По завершении доступна кнопка **«Экспорт CSV»** — таблица с одним рядом
   на спектр: имя файла, статус, top-1 группа, top-1 кандидат, score.

---

## 7. Настройки (UC-07)

Страница «Настройки» позволяет персонализировать обработку:
- **Top-K** по умолчанию для идентификации.
- **Включать Grad-CAM** по умолчанию.
- **Порог классификации** — единый порог 0.0–1.0 (используется, если
  модель не содержит per-class порогов).
- Параметры предобработки: baseline-метод (AsLS / отключено), нормализация
  (SNV / Min-Max), окно Savitzky–Golay, степень полинома.
- Язык интерфейса (рус / англ).

Настройки сохраняются в `localStorage` браузера; разные браузеры/устройства
имеют независимые настройки.

## 8. Расшифровка сообщений об ошибках

| Код | Что это значит | Что делать |
|-----|----------------|-----------|
| `PARSING_ERROR` | Файл не соответствует формату (битый JCAMP/CSV) | Проверьте структуру файла; сообщение указывает поле/строку |
| `VALIDATION_ERROR` | Содержимое спектра некорректно (NaN, размеры) | Перегенерируйте файл из исходного прибора |
| `ENTITY_NOT_FOUND` | Запрашиваемое соединение/запрос отсутствует | Проверьте корректность ID |
| `DOMAIN_ERROR` | Внутренняя ошибка (модель/БД недоступна) | Сообщите администратору, проверьте логи |
| HTTP 400 | Неверные параметры запроса (top_k вне диапазона и т. п.) | Скорректируйте параметры |
| HTTP 413 | Превышен лимит размера файла | Разбейте файл; одиночный лимит 50 МБ, батч 500 МБ |

## 9. Ограничения фазы 1 (синтетика)

Текущая версия (`v1.0-prototype`) обучена на синтетическом датасете и
служит для отладки конвейера. Точность идентификации **не достигает**
целевых значений главы 11. Реальные значения метрик появятся в фазе 2
после обучения на NIST/SDBS/Coblentz (этапы 18–21
`DEVELOPMENT_PLAN.md`).

Что это означает на практике:
- Результаты идентификации могут содержать ошибки.
- Список кандидатов формируется из 200–500 синтетических соединений.
- Объяснения Grad-CAM могут указывать на нехарактерные области спектра.

Используйте систему как **демонстрационный прототип**, а не как
производственный инструмент анализа.
