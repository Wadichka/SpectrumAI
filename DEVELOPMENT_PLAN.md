# План разработки для Claude Code

Этот файл — пошаговая инструкция, как реализовать систему распознавания органических соединений по ИК-спектрам с помощью Claude Code. Каждый этап — отдельная сессия (1–3 часа работы), с готовым промптом, критериями приёмки и коммитом в конце.

## Как пользоваться

1. **Начало сессии:** запусти Claude Code в корне проекта. Он автоматически прочитает `CLAUDE.md`.
2. **Перед каждым этапом:** скопируй промпт из соответствующего раздела ниже.
3. **Режим планирования:** перед запуском промпта переключись в Plan Mode (`Shift+Tab` дважды). Получи план — согласуй — нажми «Approve».
4. **После этапа:** прогони `критерии приёмки`, попроси коммит, начни новую сессию командой `/clear` для следующего этапа.
5. **Если этап разваливается на подзадачи** — раздели его сам. Длинные этапы (>3 ч работы) точно стоит дробить.

> Промпты ниже намеренно конкретные. Если ты добавишь «и сделай красиво» — результат будет хуже. Если хочешь отойти от схемы — отредактируй промпт, не дописывай его в чат поверх.

## Этап 3. Предобработка спектров

**Цель:** приведение спектра к единой сетке, baseline correction, нормирование.

**Промпт для Claude Code:**

> Прочитай главу 5, раздел 5.4 (схема разметки — не нужен сейчас) и главу 6, раздел 6.2.3 (целевая длина вектора). Реализуй в `backend/app/preprocessing/`:
> 1. `resample.py` — интерполяция входного спектра на сетку 400–4000 см⁻¹ шагом 1 см⁻¹ (`scipy.interpolate.interp1d`, кубический сплайн). Обрабатывай случай, когда исходный диапазон не покрывает целевой — заполнять нулями за пределами с предупреждением в логи.
> 2. `baseline.py` — Asymmetric Least Squares (AsLS) baseline correction. Реализуй из scipy.sparse, без внешних зависимостей.
> 3. `smoothing.py` — Savitzky-Golay сглаживание (`scipy.signal.savgol_filter`).
> 4. `normalize.py` — две функции: min-max нормирование и Standard Normal Variate (SNV).
> 5. `pipeline.py` — функция `preprocess(raw: RawSpectrum, config: PreprocessConfig) -> ProcessedSpectrum`, последовательно применяющая ресемплинг → baseline → smoothing → нормирование. Параметры берёт из `PreprocessConfig` (Pydantic).
>
> Тесты на синтетических спектрах: гауссовы пики на разных позициях, проверь что после ресемплинга длина = 3601 и крайние точки имеют ожидаемые волновые числа; что baseline correction убирает добавленный линейный фон; что нормированный спектр имеет ожидаемые статистики.

**Критерии приёмки:**
- Все unit-тесты предобработки зелёные.
- На синтетическом спектре с линейным фоном и пиками после `preprocess()` фон убран, длина вектора = 3601.
- Конфиг полностью настраиваемый через `PreprocessConfig`.

**Коммит:** `feat(preprocessing): ресемплинг, baseline AsLS, сглаживание, нормирование`.

**Цель:** скрипты сбора и разметки датасета, готовый `Dataset` для обучения.

> 2. `pipelines/dataset.py` — `class SpectraDataset(torch.utils.data.Dataset)`, читающий заранее подготовленный parquet-файл с колонками `spectrum` (массив 3601), `labels` (массив 0/1 длины числа групп), `smiles`, `compound_id`. Поддерживает аугментации.
> 3. `pipelines/augmentations.py` — реализация аугментаций из главы 5.7 (шум, сдвиг, масштабирование, синтетический baseline).
> 4. `scripts/build_synthetic_dataset.py` — скрипт, который генерирует **демонстрационный синтетический датасет** на 200–500 молекул (выбери разнообразный набор SMILES, можно из встроенного списка), для каждой создаёт «фейковый» спектр из суммы гауссов на характеристических частотах из таблиц функциональных групп (это не имитация физики, а заглушка для отладки пайплайна). > Важно: загрузку реальных NIST/SDBS/Coblentz не реализуем сейчас (это отдельный этап и требует данных). Синтетический датасет нужен только чтобы конвейер обучения был отлаживаемым.

> Тесты: проверь, что `label_functional_groups("CCO")` возвращает корректный вектор (этанол → гидроксильная группа), что `SpectraDataset` правильно отдаёт батчи нужной формы.

**Критерии приёмки:**
- Скрипт `python ml/scripts/build_synthetic_dataset.py` создаёт parquet-файл.
- `SpectraDataset` отдаёт тензоры формы `(3601,)` и `(n_groups,)`.

## Этап 6. Двухбашенная схема и контрастное обучение

**Цель:** спектральная и молекулярная башни + InfoNCE loss + FAISS-индекс.

**Промпт для Claude Code:**

> Прочитай главу 6, разделы 6.6 (контрастное обучение, двухбашенная архитектура), 6.7 (молекулярные энкодеры). Реализуй:
> 1. `ml/pipelines/models/spectrum_tower.py` — спектральная башня. Заверни энкодер из CNN (без классификационной головы) + проекционная голова (MLP) в `nn.Module`.
> 2. `ml/pipelines/models/molecule_tower.py` — молекулярная башня на основе предобученной ChemBERTa (`seyonec/ChemBERTa-zinc-base-v1` или аналог из главы 6.7.1). Стратегия — заморозка с трейнингом проекционной головы (раздел 6.7.3).
> 3. `losses.py` — symmetric InfoNCE loss согласно разделу 6.6.4. Реализуй с температурой τ как параметром.
> 4. `training/train_contrastive.py` — цикл совместного обучения двух башен. Поддержи multi-task режим (раздел 6.6.6): классификация + контрастная потеря с весами α, β.
> 5. `scripts/build_faiss_index.py` — после обучения проходит по всей референсной базе, считает молекулярные эмбеддинги, строит FAISS-индекс (`IndexFlatIP` для inner product), сохраняет в `models/faiss/`. Сохраняет также маппинг index_id → compound_id в JSON.
> 6. `ml/pipelines/retrieval.py` — функция `find_top_k(spectrum_embedding, k=10) -> list[CompoundCandidate]`.
>
> Тесты: forward pass обеих башен, форма выходных эмбеддингов; InfoNCE loss даёт ожидаемое значение на синтетических парах; ретривал возвращает k кандидатов.

**Критерии приёмки:**
- Два пути обучения работают (классификация и контрастное).
- FAISS-индекс строится без ошибок и поиск возвращает результаты.
- Загрузка ChemBERTa через `transformers` работает offline после первого скачивания (`local_files_only=True`).

**Коммит:** `feat(ml): двухбашенная схема, InfoNCE, FAISS-ретривал`.

## Если что-то идёт не так

- **Claude Code «уплыл» от ТЗ.** Останови его, скажи: «Сверь текущую реализацию с главой N, раздел M, и перечисли расхождения. Не правь, пока не подтвержу».
- **Контекст переполнен и качество падает.** Запусти `/clear` и начни этап заново с того же промпта.
- **Меняется требование.** Сначала обнови соответствующую главу `.docx` (формально, как в ВКР), потом — `CLAUDE.md`, потом этот файл, и только потом — код. Иначе расхождение с пояснительной запиской утечёт в защиту.
- **Нужна помощь с самой пояснительной запиской.** Это отдельный поток работы — лучше делать в Cowork-сессии (то, где Вы сейчас), а не в Claude Code.
