# Нагрузочные тесты Locust

Сценарии нагрузочного тестирования для SpectrumAI согласно главе 11 §11.6.5
и таблице 11.4 пояснительной записки.

## Сценарии

- **SingleInferenceUser** (weight=3): POST `/api/v1/identify` с одиночным JCAMP-DX,
  задержка 1–3 с между запросами. Целевая метрика — 95-й перцентиль времени
  отклика ≤ 2,0 секунд (NFR-01).
- **BatchUser** (weight=1): POST `/api/v1/identify/batch` с 10 спектрами в
  multipart, интервал 5–15 секунд. Целевая метрика — throughput ≥ 20
  спектров/секунду.

## Запуск

Перед прогоном поднимите стек:

```bash
docker compose up -d
```

Дождитесь `docker compose ps` показывая `(healthy)` для backend, затем:

```bash
cd backend
locust -f tests/load/locustfile.py \
    --host http://localhost:8000 \
    --headless \
    -u 10 -r 2 -t 60s \
    --html ../docs/test-report/locust/report.html \
    --csv ../docs/test-report/locust/stats
```

Параметры:

| Флаг | Значение | Назначение |
|------|----------|------------|
| `-u 10` | 10 пользователей | Имитирует ~10 параллельных аналитиков |
| `-r 2` | 2 юзера/сек | Постепенный «разогрев» |
| `-t 60s` | 60 секунд | Длительность стационарной фазы |
| `--html` | HTML-отчёт | Сводка по перцентилям и графики |
| `--csv` | CSV-сырьё | Для повторной агрегации/визуализации |

Для интерактивного веб-UI (анализ во время прогона): уберите `--headless` и
откройте http://localhost:8089.

## Критерии приёмки (таблица 11.4)

| Метрика | Целевая | Приёмочная |
|---------|---------|------------|
| 95p latency `/identify` | ≤ 1,5 с | ≤ 2,0 с |
| Throughput `/identify/batch` | ≥ 50 спектров/сек | ≥ 20 спектров/сек |
| Доля успешных ответов | 100% | ≥ 95% |

Отчёты сохраняются в `docs/test-report/locust/` и включаются в финальный
отчёт о тестировании (Этап 16 G).
